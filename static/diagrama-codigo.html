API Backend (Python / Flask) — app.py

Autenticação
«rotas Flask»
Rotas de Autenticação
[/login, /cadastro, /logout, /me, /esqueci-senha, /redefinir-senha]
Recebe credenciais, registra usuário e gerencia recuperação de senha.
Autentica e inicia sessão
«lógica inline»
Controle de Sessão
[Flask-Login / login_user() / logout_user()]
Login com sessão server-side via cookie seguro. Decorator @login_required protege rotas privadas.
Hash e verifica senha
«utilitário»
Hashing de Senha
[Werkzeug — generate_password_hash / check_password_hash]
Armazena senhas com hash seguro. Sem tokens JWT ou OAuth2 externo.
Usuários e Produtos
«rotas Flask»
Rotas de Usuários
[/me, /usuarios/perfil, PUT /me]
CRUD de perfil, foto de perfil, chave Pix e controle de papel (membro / admin).
«rotas Flask»
Rotas de Produtos
[GET/POST /produtos, GET/PUT/DELETE /produtos/item]
Listagem com filtros por categoria, condição e preço. Subquery SQL para rating médio e total de avaliações por produto.
Upload de foto do produto
«lógica inline»
Upload de Imagens
[POST /upload — Werkzeug + uuid + filesystem local]
Recebe multipart/form-data, gera nome único com uuid, salva em /static/assets/uploads/. Múltiplas fotos via tabela imagens_produto.
Negociação e Pedidos
«rotas Flask»
Rotas de Propostas
[GET/POST /propostas, PUT /propostas]
Comprador envia proposta com quantidade e valor. Vendedor aceita ou recusa. Preço negociado salvo em itens_carrinho.preco_negociado.
Proposta aceita → adiciona ao carrinho
«rotas Flask»
Rotas de Carrinho e Pedidos
[POST /carrinho, GET/POST /pedidos, PUT /pedidos/status]
Adiciona ao carrinho, finaliza compra e registra pedido. Vendedor atualiza status (pendente → entregue). Sem integração com gateway de pagamento externo.
Contagem de vendidos
«lógica inline»
Cálculo de Vendidos
[SQL — COUNT pedidos WHERE status='entregue']
Total de unidades vendidas calculado por query SQL na tabela pedidos. Exibido na página de detalhes do produto.
Avaliações, Notif. e Favoritos
«rotas Flask»
Rotas de Avaliações
[POST /avaliacoes, GET /avaliacoes]
Comprador avalia produto após entrega (nota 1–5 + comentário). Média por produto calculada via AVG() em subquery SQL. Distinção entre rating do produto e do vendedor.
«rotas Flask»
Rotas de Notificações
[GET/PUT /notificacoes]
Notificações persistidas em tabela MySQL (sem fila assíncrona, sem Celery/Redis). Badge com contador de não lidas. Marcação em lote como lidas.
«rotas Flask»
Rotas de Favoritos
[GET/POST/DELETE /favoritos]
Adiciona e remove produtos dos favoritos do usuário autenticado. Sincronizado com localStorage no frontend.
Administração
«rotas Flask»
Rotas Administrativas
[/admin/usuarios, /admin/produtos, /admin/pedidos]
Acessíveis apenas para usuarios.papel = 'admin'. Gestão de usuários, moderação de anúncios e visão geral de pedidos.
Verifica papel do usuário
«lógica inline»
Controle de Acesso por Papel
[current_user.papel == 'admin']
Verificação inline do campo papel na tabela usuarios. Sem biblioteca de permissões externa (sem Flask-Principal ou similar).
«lógica inline»
Estatísticas do Painel
[SQL — COUNT / SUM por tabela]
Contagem de usuários, produtos, pedidos e receita total via queries SQL diretas. Exibidas nos cards do dashboard admin.
Infraestrutura Compartilhada — usada por todas as rotas
«infraestrutura»
Conexão com Banco de Dados
[PyMySQL / DictCursor / autocommit=True]
Helper get_db() retorna conexão PyMySQL. Queries SQL puras via cursor.execute(). Resultados como dicionários (DictCursor). Sem ORM, sem migrations automáticas.
«infraestrutura»
Gerenciamento de Sessão
[Flask-Login / user_loader / @login_required]
Sessões server-side com cookie seguro. user_loader carrega usuário do banco por ID. Rotas protegidas com decorator @login_required. Sem JWT, sem OAuth2.
«infraestrutura»
CORS e Configurações
[Flask-CORS / variáveis de ambiente]
CORS habilitado para comunicação frontend ↔ backend. Credenciais do banco e secret key configuradas via variáveis de ambiente no servidor.
