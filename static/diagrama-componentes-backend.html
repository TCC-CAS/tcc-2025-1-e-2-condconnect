Diagrama de Componentes — Backend CondConnect
API Backend [container]
API Backend (Python / Flask)
[app.py — arquivo único com todas as rotas REST]
Infraestrutura de dados
«component»
Camada de Acesso a Dados
[PyMySQL / DictCursor / SQL puro — sem ORM]
Executa queries SQL diretamente via PyMySQL com DictCursor (resultados como dicionário). Conexão configurada com autocommit=True. Banco MySQL externo em thyagoquintas.com.br, banco engenharia_16.
Segurança e usuários
«component»
Autenticação e Sessões
[Flask-Login / Werkzeug / Cookies]
Login, logout e controle de sessão server-side via cookies seguros. Hashing de senha com Werkzeug. Rotas: /login, /logout, /cadastro, /esqueci-senha, /redefinir-senha.
«component»
Gerenciamento de Usuários
[Módulo de Serviço]
CRUD de usuários, edição de perfil, foto de perfil, chave Pix e controle de papel (membro / admin). Rota: /me, /usuarios/perfil.
Marketplace
«component»
Gerenciamento de Produtos
[Módulo de Serviço]
Cadastro, edição, remoção e listagem de produtos. Filtros por categoria, condição e faixa de preço. Subquery SQL para rating médio e contagem de avaliações por produto.
«component»
Upload de Imagens
[Werkzeug / uuid / Sistema de Arquivos Local]
Recebe imagens via multipart/form-data, gera nome único com uuid e salva em /static/assets/uploads/ no servidor. Suporte a múltiplas fotos por produto (tabela imagens_produto).
«component»
Favoritos
[Módulo de Serviço]
Adicionar e remover produtos dos favoritos do usuário. Sincronizado com localStorage no frontend. Rotas: GET/POST/DELETE /favoritos.
«component»
Sistema de Propostas
[Módulo de Serviço]
Comprador envia proposta com quantidade e valor. Vendedor aceita ou recusa. Preço negociado registrado em itens_carrinho.preco_negociado. Tabela propostas.
Pedidos e pós-venda
«component»
Gestão de Pedidos e Carrinho
[Módulo de Serviço]
Adição ao carrinho (tabela itens_carrinho), finalização de compra, histórico de pedidos e atualização de status (pendente → entregue). Total de unidades vendidas via COUNT em pedidos.
«component»
Avaliações
[Módulo de Serviço]
Comprador avalia produto após entrega (nota 1–5 + comentário). Média calculada por produto via subquery SQL. Distinção entre rating do produto e rating geral do vendedor.
«component»
Notificações Internas
[Módulo de Serviço / tabela MySQL]
Notificações persistidas em tabela notificacoes no banco. Sem fila assíncrona (sem Celery/Redis). Badge com contador de não lidas. Rotas: GET/PUT /notificacoes.
